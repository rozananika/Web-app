Library Management System - System Design

Use Case Diagram

The system has four main user roles with different permissions and capabilities:

  • Guest User: Can browse books, view details, and register
  • Member: Can borrow books, write reviews, and get personalized recommendations
  • Librarian: Can manage books, handle overdue cases, and generate reports
  • Admin: Can manage users, assign roles, and monitor system usage

Sequence Diagram - Book Borrowing Process

The sequence diagram illustrates the book borrowing process, which includes:

  • Book Search & Selection: User searches and selects a book
  • Book Borrowing: System checks availability and processes the loan
  • Analytics Update: System updates user's borrowing statistics
